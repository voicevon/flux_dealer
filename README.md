# sorter_mini_controller (ESP32)

## 项目简介

本项目是为 **8 级线性级联式芦笋分拣机** 深度定制开发的嵌入式控制器固件。它运行于 **ESP32** 硬件平台，基于 **PlatformIO + Arduino Framework** 进行构建。

分拣控制的核心业务逻辑（分拣决策层）由手机端/上位机应用 (`sorter_mini_phone`) 承担，固件通过 **蓝牙低功耗 (BLE)** 与其通信，接收每次判定出的目标槽位 ID (Target ID 1~8)。在物理层面，ESP32 采用**“共享脉冲同步驱动”与“串行移位 I/O 扩展”**的高可靠性硬解耦架构，控制 8 路步进电机及对应的归零限位传感器，实现多节拍、流水线式的自动同步分拣。

---

## 机械结构与分拣原理

分拣机构共有 8 级翻转板（分别由步进电机 M0 至 M7 驱动），物理排布为线性级联排列：

1. **物料入口**：
   物料（如一根芦笋）放置在入口处。当红外落料传感器（`ENTRANCE_SENSOR_PIN`）检测到物料落入时，触发系统进入一个**新节拍 (New Beat)**。

2. **多级流水线逻辑 (Pipeline)**：
   * 系统内部维护一个长度为 8 的逻辑流水线 `_pipeline[8]`。
   * 新节拍开始时，流水线整体后移一格，从蓝牙队列中取出最新的 Target ID 填入首级（M0 对应 `_pipeline[0]`）。
   * 每一级翻转机构在每个节拍只需做出三种动作之一：
     * **放行 (Forward, Action: 1)**：翻转板正转，将物料导向下一级滑道。
     * **剔除 (Reverse, Action: -1)**：翻转板反转，将物料直接滑落入该级对应的本地物料框（终点）。
     * **不动作 (Neutral, Action: 0)**：翻转板保持中立位置（不激活电机）。

3. **分拣路由表 (`ROUTING_TABLE`)**：
   路由逻辑通过二维静态矩阵实现，实现业务逻辑与硬件代码的完全解耦：
   * **Target 1**：在 M0 处剔除（滑入1号槽），后续级不动作。
   * **Target 2**：在 M0 处放行，在 M1 处剔除（滑入2号槽）。
   * ...
   * **Target 8**：在 M0~M6 处放行，在 M7 处剔除（滑入8号槽）。
   
每个节拍运动完成后，系统均执行 `setCurrentPosition(0)` 锁步，由于分拣轮具有 90° 旋转对称性，可完全消除机械累积误差。

## 硬件架构与引脚定义 (ESP32)

为在极少引脚开销下支持 8 路电机，系统采用了如下硬件架构：

### 1. 共享脉冲 + 独立屏蔽（A4988 步进驱动）
* **共享脉冲 (Shared STEP)**：所有 8 个电机驱动器的 STEP 脚硬件并联，接至 ESP32 的 `GPIO 5`。系统通过一个全局 `AccelStepper` 实例产生统一的加减速脉冲，保证多轴动作的绝对同步。
* **使能屏蔽 (ENABLE)**：使用 8 个独立的 GPIO 分别控制 8 路驱动器的 ENABLE 引脚。
  * **低电平放行 (Low)**：使能驱动器，电机跟随共享脉冲同步旋转。
  * **高电平屏蔽 (High)**：屏蔽脉冲，但电机驱动器保持通电锁步力矩，防止机构自转或受外力错位。
  * **引脚定义**（在 `pins.h` 中）：
    * `M0 -> GPIO 13`, `M1 -> GPIO 12`, `M2 -> GPIO 14`, `M3 -> GPIO 27`
    * `M4 -> GPIO 26`, `M5 -> GPIO 25`, `M6 -> GPIO 33`, `M7 -> GPIO 32`

### 2. SPI 移位寄存器总线 (DIR 与 HOME 扩展)
通过 **74HC595 (串转并输出)** 和 **74HC165 (并转串输入)** 菊花链式串联，仅占用 ESP32 的 4 个引脚便可控制 8 路电机方向 (DIR) 并读取 8 路归零限位状态 (HOME)：
* **`LATCH_PIN` (GPIO 18)**：锁存引脚 (STCP / PL)
* **`CLOCK_PIN` (GPIO 19)**：移位时钟 (SHCP / CP)
* **`DIR_DATA_OUT` (GPIO 21)**：DIR 串行数据输出 (DS)
* **`HOME_DATA_IN` (GPIO 34)**：HOME 串行数据输入 (Q7)

### 3. 传感器与指示灯
* **红外落料传感器**：`ENTRANCE_SENSOR_PIN` (GPIO 39)，支持软件级 50ms 窗口防抖。
* **板载状态指示灯**：`LED_PIN` (GPIO 2)，用于指示状态（1Hz慢闪=正常心跳，10Hz快闪=蓝牙物理辨识中，错误挂起时常亮）。

## 软件组件划分与 HAL 抽取

代码在项目 Phase 2/3 重构中引入了**硬件抽象层 (HAL)**，结构如下：
* **`hal/ShiftRegisterBus`**：封装 74HC595 和 74HC165 的全双工模拟 SPI 传输时序。
* **`hal/MotorHardware`**：封装使能屏蔽操作和共享 `AccelStepper` 脉冲的产生。
* **`hal/SensorInput`**：实现落料传感器防抖及限位回读。
* **`MotionPlanner`**：纯逻辑路由计算类，不依赖 Arduino SDK，根据流水线输入生成 `MotionCommand` 动作指令。
* **`HomingController`**：两段式归零器。快速接近限位 (800步/秒) -> 反弹脱离 -> 慢速高精度锁定 (200步/秒)，包含 10 秒超时挂载保护。
* **`BleManager`**：蓝牙服务，实现基于 MAC 地址生成唯一广播名（`Sorter_XXXXXX`）以便于扫码安全配对，并支持设备物理闪烁辨识（指令 0x03）。
* **`SorterFSM`**：核心状态机，控制从 `IDLE -> NEW_BEAT_PREP -> MOVING -> SLIDING -> COMPLETED` 的节拍流转。

## PC 本地单元测试 (Native Unit Testing)

本项目支持在非 ESP32 硬件环境（如普通的开发机 PC 上）直接编译和测试核心路由逻辑：
1. **测试原理**：`MotionPlanner` 与硬件操作完全解耦，不调用任何底层 Arduino 硬件 API。
2. **测试框架**：基于 PlatformIO `native` 环境。
3. **运行测试**：
   在开发机终端中直接运行：
   ```bash
   pio test -e native
   ```
   这可以在上机实测前验证所有路由和队列边界，避免硬件机械干涉和碰撞风险。

## 快速上手与操作说明

### 1. 开发环境安装
* 安装 [VS Code](https://code.visualstudio.com/) 并确保添加至系统 Path。
* 在 VS Code 中安装 **PlatformIO IDE** 插件。
* 用 VS Code 打开当前项目文件夹 `sorter_mini_controller`。

### 2. 编译与烧录
将 ESP32 开发板连接至电脑 USB 口：
* **编译项目**：在 VS Code 底部工具栏点击 **“✔” (Build)**，或在终端中执行 `pio run`。
* **烧录固件**：点击 **“➔” (Upload)**，或执行 `pio run --target upload`。
* **串口监视**：点击 **“🔌” (Monitor)**，或执行 `pio device monitor`（默认波特率 115200）。

### 3. 系统状态指示 (LED)
* **1Hz 慢闪**：系统正常待机/空闲，蓝牙已广播。
* **10Hz 快闪**：手机端通过蓝牙发送了物理辨识指令，LED 快速交替闪烁 4 秒以指示物理方位。
* **异常常亮**：归零超时（限位损坏或接线断路超过 10 秒），系统挂起保护。
